# D. 迷宫冒险

练习了一下折半搜索

看到数据范围 N<=35，状态总数高达 2^35，第一反应绝不能是普通的动态规划，感觉又要TLE又要MLE

又由于这道题操作的独立性与无序性：按开关的顺序不影响最终结果，且每个开关只有“按”或“不按”两种有意义的状态

因此，最完美的解法是折半搜索。我们将 35 个开关物理平分为两组：

前半组 17 个开关，暴力枚举 2^17种按法状态，并存入哈希表

后半组同样枚举，利用异或运算的自反性（A ^ B = Target 推导得 A = Target ^ B），在哈希表中以 O(1) 的时间查找互补状态。指数级的乘法化为加法

**遇到的易错点：**

位运算溢出：状态压缩时常写 `1 << i`。当 i >= 31 时，这会导致 32 位整型溢出，变成乱码。必须写成 `1LL << i` 开启 64 位空间

误用双向 BFS：双向搜索是对状态树层层扩展，中间相遇层依然面临组合数级别的海量状态，必定MLE

但直接深搜的话，最坏情况递归深度可达20 万层，必定爆栈且超时，这时候就需要用到“正难则反”的思想，利用异或特性，选 K 个人等价于“丢弃 N-K 个人”。当 K > N/2 时，我们转而去 DFS 寻找那几个被淘汰的人，最后用全体总异或和将其翻转回来，把递归深度压在安全范围内

容易错的点主要在于（交的时候踩了其中某些）：

忘记复原状态：很显然运用正难则反yi'ding'hui'dao'zhi触发反转策略后，求出的是落选者的异或和。如果结算时忘记用 `Total_XOR ^ current_xor` 还原就会错

乱用最优化剪枝： 异或运算不具备单调性（当前值小不代表未来不会暴涨），只能使用**可行性剪枝**（判断剩余人数是否还够凑齐队伍），绝对不能因为当前异或和较小就提前结束搜索

# I. 奶*

(看见标题直接一秒进了（bushi）)

题目要求对不断变化的 $a_1$ 进行多次模拟。若每次都从头跑，必定超时。破题核心在于洞察静态结构：除了 $a_1$ 以外，从 $a_2$ 到 $a_n$ 的规则和跳跃代价是永恒不变的

程序的运行轨迹本质上是一个出度永远为 1 的“有向图”。我们在图上跑带缓存的 DFS（记忆化搜索），用 `dp[type][x]` 记录从某状态出发到程序越界积累的总收益。因为 $a_1$ 一直在变，我们不能把涉及 $a_1$ 的路线存入缓存；但巧妙的是，程序的绝对起点永远是 `x=1` 且准备向前跳。一旦后续轨迹又回到这个原点，无论当前的 $a_1$ 是多少，都意味着陷入了绝对的死循环

容易错的地方：

二色标记法致死： 仅用 true/false 标记访问，无法区分“遇到了已经算好的安全节点（应取缓存）”和“遇到了当前路线的脚印（死循环）”。必须用 0, 1, 2 三色标记法。

信号量被掩盖： 发现死循环返回 `-1` 时，如果外层递归盲目将其与当前代价相加，代表死循环的信号就会被破坏。必须显式判断并拦截 `-1`

状态少开一维： 站在位置 x，向前跳和向后跳是截然不同的命运。只开一维 `dp[x]` 会导致不同方向的状态互相污染

# B. 魔法启动

明显本题要求魔力精确匹配，不能超额。由于有的石头它是奇数，分裂之后就会有。5小数出现，所以最好将目标整体翻倍为 2m，这样石头的三种选择（不用、全用、分裂一半）带来的魔力贡献就变成了纯整数（0, $2A_i$, $A_i$），方便计算

面对 N <= 30，每个石头 3 种选择，3^30 的复杂度依然会超时，我们再次祭出折半搜索，前 15 个石头搜出所有魔力组合，存入数组并去重排序；后 15 个石头边搜边查，拿到当前积的魔力后，利用二分查找去左半部分寻找精确匹配的互补值，最终合并出最小的分裂次数。

运用了字典，以及排序去重等等数据上的技巧

易错点剖析：

按值传递导致内存爆炸： 在右半边 DFS 时，如果左侧几十万大小的字典数组按值传递（未加 `&` 引用），每次递归都会引发海量内存拷贝，造成 MLE

去重时的必须留下最好的解： 在整理左半边状态时，如果多种方案都能凑出相同的魔力值，必须且只能保留分裂次数最少的那一个，否则后续合并会错失最优解

盲目信任 `lower_bound`： 该函数返回的是第一个“大于等于”目标的元素。拿到迭代器后，必须二次校验它是否越界，以及对应的值是否精确等于需要的互补魔力
