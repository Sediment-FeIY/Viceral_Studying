# F. 多项式乘法

如果顺着题目“系数表示法”的定义，每一项去乘另一项，复杂度必然是 $O(n^2)$。题解中虽然给出暴力可以过，但显然我们需要学习的并不是这种方法，故想到FFT

FFT的核心思想如下：

1. **转点值：** 利用离散傅里叶变换（DFT），挑选复数平面上的单位根作为 $x$ 代入，将两个多项式转换为点值表示法

2. **乘点值：** 相同 $x$ 下的 $y$ 值直接相乘，这一步 $O(n)$

3. **转系数：** 利用逆变换（IDFT），将乘积的点值重新变回系数表示法

   这套流程利用了单位根的对称性分治，成功将复杂度降至 $O(n \log n)$

怎么实现：

这是标准的快速傅里叶变换（FFT）模板。工程上的核心在于蝴蝶变换（二进制翻转），我们需要将递归转换为底层的迭代，先利用 `rev` 数组将元素置换到正确的位置，再一层层向上合并

**踩坑：**

- **浮点数精度截断：** C++ 处理三角函数存在微小误差，算出来的系数可能是 `2.999999`：在 IDFT 除以 `limit` 之后，输出时不能直接强转 `int`，必须写成 `(int)(val + 0.5)` 进行四舍五入补偿否则WA

# W. Faker vs Bin

（虽然是easy但是合理怀疑过的人多主要是因为这个标题（））

明显是博弈论——公平组合游戏，数据范围 $10^9$ 非常巨大意味着不能用 SG 函数打表，必须推导 $O(1)$ 的数学公式

- 如果没有 $k$（只能拿 1 或 2），是经典的巴什博弈，后手的必败态是 $3$ 的倍数（$n \equiv 0 \pmod 3$）

  **拆解如下：**

  **$n=0$：** 没得拿了，当前玩家输，它是绝对的 P 态

  **$n=1$：** 拿 1 刀，剩下 0（P 态），因为能走到 P 态，所以 1 是 N 态

  **$n=2$：** 拿 2 刀，剩下 0（P 态），因为能走到 P 态，所以 2 是 N 态

  **$n=3$：** 只能拿 1 或 2 刀

  - 拿 1 刀 $\rightarrow$ 剩下 2 (N 态)
  - 拿 2 刀 $\rightarrow$ 剩下 1 (N 态)

- 引入 $k$ 后进行分类讨论：

  - **$k$ 不是 3 的倍数：** 拿 $k$ 的效果等同于拿 1 或拿 2（在模 3 意义上），完全也没有打破原有平衡，规律不变，依然看n(mod 3)
  - **$k$ 是 3 的倍数：**原本 $k$ 是必败态，现在可以一招秒杀，变成了必胜态。推导发现，每遇到一次$nk$,就会发生一次“打断”,局面的周期从 3 变成了 k+1

解决理论推导和博弈论的理解之后，算法实现较易，仅是极其简单的 $O(1)$ 分支判断

还有这些东西要稍微注意一下：

如果是第二种情况，算出当前在周期内的位置 `rem = n % (k + 1)`。此时的必败态依然是 3 的倍数，但要把 $k$ 本身剔除

在第二种情况中，只写了 `rem % 3 == 0`，不要忘记加上 `&& rem != k`。因为 $k$ 是 3 的倍数但它可以被一次拿完，属于必胜态

# A. 还原符号

矩阵元素高达 $10^9$，$100 \times 100$ 的行列式绝对值极大，写高精度不仅代码灾难且极易超时，选择哈希映射

关键点在于取模映射。我们选取一个大质数 $P$（如 $10^9+7$）。（一开始顺手写成+5，居然还没发现，我晕）

1. 遍历题目给的绝对值字符串，算出 `abs_mod` $= |\det(A)| \pmod P$

2. 对原矩阵跑一遍模 $P$ 意义下的高斯消元，算出 `det_mod` $= \det(A) \pmod P$

   因为 $P$ 很大，碰撞概率极低。如果原符号是正，两者必定相等；如果是负，必定有 `det_mod == P - abs_mod`

核心实现：

高斯消元求行列式模板。只需记住在模意义下，传统的除法要替换成乘以主元的乘法逆元（通过费马小定理 + 快速幂求解）

**易错点：**

- **负数取模取不对：** C++ 中负数取模还是负数，读入矩阵后必须立刻格式化：`A[i][j] = (A[i][j] % P + P) % P`
- **线性代数定理部分：** 高斯消元时，如果为了找非零主元而交换了两行，行列式的符号必须取反，在代码中体现为 `det_mod = P - det_mod`
